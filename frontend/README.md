# AI Travel Planner ✈️

## Overview

AI Travel Planner is a full-stack web application that helps users create personalized travel itineraries using Generative AI. Instead of manually researching destinations, activities, hotels, and budgets, users can simply provide their travel preferences and receive a complete day-by-day itinerary generated by Google's Gemini AI.

The application supports user authentication, trip management, itinerary customization, budget estimation, hotel recommendations, and an AI-powered packing assistant. Each user's data is securely isolated so that users can only access and manage their own trips.

This project was developed as part of a Full Stack Engineering Assessment to demonstrate skills in frontend development, backend API development, database design, authentication, AI integration, and deployment.

---

## Problem Statement

Planning a trip often involves searching across multiple websites for destinations, accommodations, activities, budgets, and travel essentials. This process can be time-consuming and overwhelming.

The goal of this project is to simplify travel planning by leveraging Generative AI to automatically generate personalized travel plans based on user preferences.

## Key Features

### User Authentication

* User Registration
* User Login
* JWT-based Authentication
* Password Encryption using bcryptjs
* Protected Routes

### AI-Powered Trip Generation

Users can generate personalized travel plans by providing:

* Destination
* Number of Travel Days
* Budget Preference
* Travel Interests

The Gemini AI model generates:

* Day-wise itinerary
* Recommended activities
* Hotel suggestions
* Estimated travel budget
* Packing checklist

### Trip Management

Users can:

* Create trips
* View saved trips
* Update trip information
* Delete trips
* Manage their travel plans securely

### Budget Estimation

Each generated trip includes:

* Transportation cost
* Accommodation cost
* Food cost
* Activity cost
* Total estimated budget

### Hotel Recommendations

The system suggests hotels according to the selected budget category:

* Budget
* Medium
* Luxury

### AI Packing Assistant

A smart packing checklist is generated based on:

* Destination
* Climate
* Activities included in the itinerary

Examples:

* Passport
* Travel documents
* Raincoat
* Hiking shoes
* Sunscreen
* Power bank

### User Data Isolation

Every trip is linked to its owner.

Users cannot access:

* Other users' trips
* Other users' packing lists
* Other users' itinerary information

```text

        │
 ┌──────┴──────┐
```

## Installation and Setup

### Clone Repository

```bash
git clone https://github.com/your-username/ai-travel-planner.git

cd ai-travel-planner
```

---

### Backend Setup

Navigate to backend folder:

```bash
cd backend
```

Install dependencies:

```bash
npm install
```

Create a `.env` file:

```env
PORT=5000

MONGO_URI=your_mongodb_connection_string

JWT_SECRET=your_secret_key

GEMINI_API_KEY=your_gemini_api_key
```

Start backend server:

```bash
npm run dev
```

---

### Frontend Setup

Navigate to frontend folder:

```bash
cd frontend
```

Install dependencies:

```bash
npm install
```

Create `.env` file:

```env
VITE_API_URL=http://localhost:5000/api
```

Run frontend:

```bash
npm run dev
```

## API Endpoints

### Authentication

| Method | Endpoint           | Description   |
| ------ | ------------------ | ------------- |
| POST   | /api/auth/register | Register User |
| POST   | /api/auth/login    | Login User    |

### Trips

| Method | Endpoint       | Description       |
| ------ | -------------- | ----------------- |
| POST   | /api/trips     | Generate New Trip |
| GET    | /api/trips     | Get User Trips    |
| GET    | /api/trips/:id | Get Single Trip   |
| PUT    | /api/trips/:id | Update Trip       |
| DELETE | /api/trips/:id | Delete Trip       |

## Challenges Faced

During development, several challenges were encountered:

* Designing a structured JSON response format for Gemini AI
* Handling invalid AI responses
* Maintaining user-specific data isolation
* Managing authentication across frontend and backend
* Integrating MongoDB Atlas securely
* Handling API failures gracefully

These challenges were addressed through proper schema design, authentication middleware, error handling, and validation mechanisms.
